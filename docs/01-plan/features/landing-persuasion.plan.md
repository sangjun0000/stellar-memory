# landing-persuasion Planning Document

> **Summary**: 랜딩 페이지를 "설득 중심"으로 재구성 — 왜 필요한지, 누구를 위한 건지, 어떻게 동작하는지를 먼저 보여주고, 다운로드는 맨 마지막에 배치
>
> **Project**: stellar-memory
> **Date**: 2026-02-20
> **Status**: Draft

---

## 1. 현재 문제

현재 랜딩 페이지는 기능 나열 위주:
- Hero → Use Cases → Features → How It Works(코드 예제) → Get Started
- "How It Works"가 코드 예제(store/recall API)로 되어 있어 **개발자가 아니면 이해 불가**
- 다운로드/설치가 중간에 섞여 있어 **충분히 설득되기 전에 이탈**

## 2. 목표

방문자가 페이지를 위에서 아래로 읽으며 자연스럽게 설득되는 흐름:

```
공감 (왜 필요해?) → 대상 (나한테 해당돼?) → 동작 방식 (어떻게 되는 거야?) → 설치 (해볼래!)
```

## 3. 새로운 페이지 구조

### Section 1: Hero (유지, 메시지 강화)
- 핵심 메시지: "AI가 당신을 기억합니다"
- 부제: "매번 같은 설명을 반복하고 계신가요? Stellar Memory를 설치하면 AI가 대화를 기억합니다"
- CTA: "어떻게 동작하나요?" (아래로 스크롤 유도, 다운로드 버튼 아님)

### Section 2: 왜 필요한가 (Pain Point) — 신규
- **문제 공감 섹션**
- 3가지 pain point 카드:
  1. "매번 처음부터 설명" — AI에게 같은 배경 정보를 반복 입력
  2. "대화가 끊기면 끝" — 새 채팅 = 백지 상태
  3. "내가 누군지 모르는 AI" — 취향, 스타일, 맥락 모름
- 각 카드에 before/after 비교 (말풍선 형태)

### Section 3: 누구를 위한 건가 (기존 Use Cases 개선)
- 기존 4개 페르소나 유지하되, 각각에 **구체적 시나리오** 추가:
  - 학생: "지난주 공부한 내용을 AI가 이어서 설명"
  - 작가: "캐릭터 설정을 매번 안 알려줘도 AI가 기억"
  - 직장인: "프로젝트 맥락을 AI가 누적 파악"
  - 개발자: "코드 컨벤션, 아키텍처를 AI가 학습"

### Section 4: 어떻게 동작하나 — 핵심 변경
- **코드 예제 제거**, 일반인이 이해할 수 있는 설명으로 교체
- 3단계 시각적 설명:
  1. **설치** — "당신이 쓰는 AI(Claude, ChatGPT, Cursor 등)에 Stellar Memory를 연결합니다" (아이콘: AI 로고들 + 플러그 연결)
  2. **자동 기억** — "대화하면 AI가 중요한 정보를 자동으로 기억합니다. '기억해'라고 말하지 않아도 됩니다" (아이콘: 뇌 + 별)
  3. **기억 활용** — "다음 대화에서 AI가 당신을 알고 있는 상태로 시작합니다" (아이콘: 대화 말풍선 + 체크)
- 핵심 메시지: **"기존 AI에 기억력을 추가하는 플러그인입니다. AI를 바꿀 필요 없습니다."**

### Section 5: Features (기존 유지, 위치만 이동)
- 5-Zone Orbit, Self-Learning, Multimodal 등
- 관심 있는 사용자를 위한 기술적 차별점

### Section 6: Get Started / 다운로드 (맨 하단)
- 기존 설치 위자드 유지 (Claude/Cursor/ChatGPT/Other)
- 위치만 페이지 최하단으로 이동
- CTA 강조: "지금 무료로 시작하기"

## 5. 변경하지 않는 것

- 설치 위자드 기능 (그대로 유지, 위치만 이동)
- 다국어 지원 구조 (i18n 키 추가만)
- 네비게이션 바, 푸터
- 기존 CSS 변수/테마

## 6. 성공 기준

- 비개발자가 읽어도 "이게 뭔지, 왜 필요한지" 이해 가능
- 코드 없이 동작 방식 전달
- 다운로드는 충분히 설득된 후 최하단에서 진행
