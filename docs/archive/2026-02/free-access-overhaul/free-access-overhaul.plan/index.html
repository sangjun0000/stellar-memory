    Plan: free-access-overhaul

    1. 개요

    항목 내용
    Feature free-access-overhaul
    목표 스텔라 메모리를 100% 무료로 전환하고, 프로그래밍을 전혀 모르는 사람도 쉽게 사용할 수 있도록 접근성 대폭 개선
    핵심 전략 시장 장악 우선 → 수익화는 나중에. 무료 배포로 사용자 기반 확보
    사용자 선택지 로컬 모드 (내 컴퓨터에서 실행) 또는 클라우드 모드 (웹에서 바로 사용)

    2. 전략적 방향

    2.1 핵심 원칙

    ┌─────────────────────────────────────────────────┐
│            Stellar Memory 전략 전환              │
├─────────────────────────────────────────────────┤
│                                                  │
│  BEFORE (현재)          AFTER (변경 후)           │
│  ────────────          ─────────────            │
│  Free/Pro/ProMax 3계층   100% 무료, 제한 없음     │
│  결제 시스템 운영         결제 시스템 제거          │
│  개발자 대상              누구나 (비개발자 포함)    │
│  pip install 필수         1클릭 설치 or 웹 접속    │
│  CLI 명령어 숙지 필요     GUI + 자동화             │
│                                                  │
│  수익화 시점: 시장 장악 후 (추후 별도 계획)        │
└─────────────────────────────────────────────────┘

    2.2 사용자 여정 (Before vs After)

    Before (현재): 

    사용자 → pip install → CLI 명령어 학습 → 코드 작성 → 기억 저장/검색
         (파이썬 필요)  (터미널 필요)    (프로그래밍 필요)

    After (변경 후): 

    사용자 → 2가지 선택:
  ├─ [로컬] 설치 파일 다운로드 → 더블클릭 실행 → 바로 사용
  │         (프로그래밍 지식 불필요)
  │
  └─ [클라우드] 웹사이트 접속 → 회원가입 → 바로 사용
              (설치 불필요)

    3. 현재 상태 분석

    3.1 제거/비활성화 대상 (수익화 관련)

    항목 파일 현재 상태 변경
    결제 시스템 stellar_memory/billing/ 3개 결제사 연동 비활성화 (코드 보존)
    티어 제한 stellar_memory/billing/tiers.py Free 1,000개 제한 제한 없음 (무제한)
    가격표 UI landing/index.html 가격 섹션 3티어 비교표 제거, "무료" 강조
    결제 링크 landing/index.html checkout URL Lemon Squeezy 링크 제거
    Schema.org landing/index.html JSON-LD Offer 가격 정보 무료로 변경
    Fly.io 빌링 fly.toml BILLING_ENABLED=true false
    API 인증 stellar_memory/auth.py API Key 필수 로컬: 불필요, 클라우드: 간소화

    3.2 개선 대상 (접근성)

    항목 현재 문제 목표
    설치 방법 pip install 파이썬/터미널 필요 1클릭 설치 파일
    사용 방법 CLI + 코드 작성 프로그래밍 지식 필요 GUI 또는 웹 인터페이스
    문서 개발자용 README 비개발자 이해 불가 그림 + 단계별 가이드
    MCP 설정 init-mcp CLI 명령 터미널 사용 필요 자동 감지 + 1클릭
    언어 영어 중심 비영어권 장벽 다국어 (이미 5개국어 지원)

    4. 변경 계획

    F1. 수익화 시스템 비활성화 (무료 전환)

    목표: 모든 제한 해제, 결제 UI 제거, 전 기능 무료 개방

    항목 변경 내용
    billing/tiers.py 모든 티어 제한을 무제한으로 변경 (max_memories=-1, 무제한)
    config.py BillingConfig.enabled 기본값 False로 변경
    fly.toml STELLAR_BILLING_ENABLED=false
    landing/index.html 가격 섹션 → "무료 & 오픈소스" 섹션으로 교체
    landing/index.html Lemon Squeezy checkout 링크 제거
    landing/index.html Schema.org JSON-LD에서 가격 정보 → "Free"
    README.md 가격 관련 문구 제거
    Rate limiting 티어 기반 제한 → 기본 Rate limit만 유지 (서버 보호용)

    중요: 빌링 코드 자체는 삭제하지 않고 비활성화만 한다 (추후 수익화 시 재활용).

    F2. 로컬 모드 간소화 (비개발자용 1클릭 설치)

    목표: 프로그래밍을 모르는 사람이 설치 파일 더블클릭만으로 사용

    F2.1 원클릭 설치 스크립트

    운영체제별 설치 방법:

[Windows] stellar-memory-setup.bat
  → Python 자동 설치 확인 → pip install → MCP 자동 설정 → 완료 메시지

[macOS/Linux] stellar-memory-setup.sh
  → Python 확인 → pip install → MCP 자동 설정 → 완료 메시지

    설치 스크립트 동작: 1. Python 설치 여부 확인 (없으면 설치 안내 링크 제공) 2. pip install stellar-memory[mcp] 자동 실행 3. Claude Desktop / Cursor 자동 감지 4. MCP 설정 자동 적용 5. "설치 완료! AI에게 '기억해줘'라고 말해보세요" 메시지 출력

    F2.2 간소화된 CLI (비개발자 친화)

    현재 CLI 명령어가 너무 많고 복잡함. 핵심만 남기고 자동화:

    # 현재 (복잡)
stellar-memory serve --mcp --transport stdio
stellar-memory init-mcp --ide claude --yes

# 변경 후 (간단)
stellar-memory setup    # 전부 자동 (감지 + 설치 + 설정)
stellar-memory start    # MCP 서버 시작 (자동 감지)
stellar-memory status   # 상태 확인

    F2.3 로컬 사용 시나리오

    비개발자 사용 흐름:
1. stellar-memory-setup.bat 다운로드 & 실행
2. "설치가 완료되었습니다. Claude Desktop을 열어주세요"
3. Claude에게: "내 이름은 김철수야. 기억해줘"
4. Claude: "네, 김철수님! 기억했습니다."
5. (다음 날) Claude에게: "내 이름 뭐야?"
6. Claude: "김철수님이시죠!"

→ 프로그래밍 지식 제로로 AI 기억 기능 사용

    F3. 클라우드 모드 (웹에서 바로 사용)

    목표: 설치 없이 웹 브라우저만으로 스텔라 메모리 사용

    F3.1 클라우드 서비스 구조

    사용자 → stellar-memory.com/app → 회원가입(이메일) → 바로 사용

클라우드 제공 기능:
├─ 웹 대시보드: 기억 저장/검색/관리 (GUI)
├─ API 엔드포인트: 개발자용 REST API (api.stellar-memory.com)
├─ MCP 엔드포인트: AI IDE에서 원격 연결 (mcp.stellar-memory.com)
└─ 데이터: PostgreSQL + pgvector (서버에서 관리)

    F3.2 클라우드 무료 정책

    항목 정책
    가격 완전 무료
    기억 저장 수 무제한 (공정 사용 정책 적용)
    API 호출 Rate limit만 적용 (서버 보호)
    데이터 보관 무기한
    회원가입 이메일만 (소셜 로그인은 추후)

    F3.3 클라우드 vs 로컬 비교표 (사용자 안내용)

    F5.1 README.md 구조 변경

    # Stellar Memory - AI가 당신을 기억합니다

> 무료 · 오픈소스 · 30초 설치

## 스텔라 메모리란?
AI에게 기억 능력을 부여하는 도구입니다.
한번 말한 것을 AI가 다음에도 기억합니다.

## 시작하기 (2가지 방법)

### 방법 1: 내 컴퓨터에서 (로컬)
1. [설치 파일 다운로드](링크)
2. 파일 실행
3. 끝! AI에게 "기억해줘"라고 말해보세요

### 방법 2: 클라우드에서 (웹)
1. [stellar-memory.com](링크) 접속
2. 무료 가입
3. 끝! 바로 사용 가능

## 개발자라면
pip install stellar-memory
(상세 개발자 문서는 docs.stellar-memory.com 참조)

    7. 아키텍처 변경

    변경 전:
사용자 → pip install → CLI → 코드 작성 → 결제(Pro/ProMax)

변경 후:
사용자 → 2가지 선택:
  │
  ├─ [로컬]
  │   ├─ stellar-memory-setup.bat/sh 다운로드
  │   ├─ 실행 → Python 확인 → pip install 자동
  │   ├─ Claude/Cursor 자동 감지 → MCP 설정
  │   └─ 완료! AI가 기억 시작
  │
  └─ [클라우드]
      ├─ stellar-memory.com/app 접속
      ├─ 이메일 가입 (무료)
      ├─ 웹 대시보드 or MCP 원격 연결

    8. 리스크 및 대응

    리스크 대응
    무료 사용자 대량 유입 시 서버 비용 로컬 모드 적극 홍보 (서버 부담 분산)
    클라우드 남용 (스팸 가입) Rate limit + 공정 사용 정책으로 방어
    수익 없이 서버 운영비 초기에는 최소 스펙으로 운영 (Fly.io free tier 활용)
    비개발자가 Python 설치 실패 설치 스크립트에 단계별 안내 + FAQ
    기존 유료 사용자 현재 유료 사용자 거의 없음 (출시 초기). 전원 무료 전환
    추후 수익화 전환 시 반발 기존 무료 기능은 유지, 프리미엄 기능만 유료화

    • [ ] 랜딩 페이지에서 가격표 완전 제거, 로컬/클라우드 선택 UI 표시
    • [ ] 비개발자가 설치 파일 다운로드 → 실행만으로 로컬 설정 완료
    • [ ] 클라우드 모드 안내 페이지 존재 (가입 → 바로 사용)
    • [ ] README가 비개발자도 이해 가능한 수준
    • [ ] stellar-memory setup 한 줄로 전체 설정 완료
    • [ ] 결제/빌링 코드는 삭제하지 않고 비활성화 상태 보존

    10. 수익화 로드맵 (추후 - 시장 장악 후)

    현재 단계 (이번 작업):
  → 100% 무료 배포, 사용자 기반 확보

미래 단계 (시장 장악 후):
  → 기존 무료 기능은 영구 무료 유지
  → 프리미엄 기능 추가 (엔터프라이즈 전용):
    - 팀 공유 기억 (다수 사용자 동기화)
    - SLA 보장 (99.9% 가용성)
    - 전용 서버 (데이터 격리)
    - 우선 지원
  → 결제 시스템 재활성화 (billing/ 코드 재사용)
